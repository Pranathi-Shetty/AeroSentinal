## Real Datasets

### 1. NASA C-MAPSS (Commercial Modular Aero-Propulsion System Simulation)

| Field | Value |
|---|---|
| **Subsets Used** | FD001 (single operating condition, single fault mode), FD004 (six operating conditions, two fault modes) |
| **Original Source** | NASA Prognostics Center of Excellence Data Repository |
| **Current Access** | Kaggle mirrors (official NASA portal under review as of 2026) |
| **Download URL** | `https://www.kaggle.com/datasets/behrad3d/nasa-cmaps` or GitHub mirror |
| **License** | NASA Open Data — public domain, freely redistributable |
| **Citation** | Saxena, A., Goebel, K., Simon, D., & Eklund, N. (2008). "Damage propagation modeling for aircraft engine run-to-failure simulation." ICPHM 2008. |
| **Nature** | Simulated degradation data from a physics-based engine model — NOT real flight recorder data. C-MAPSS is itself a high-fidelity simulation, which is standard and accepted in PHM research. |
| **Size** | FD001: ~20,631 rows (100 engines), FD004: ~61,249 rows (249 engines) |

### 2. UCI Condition Monitoring of Hydraulic Systems

| Field | Value |
|---|---|
| **Source** | UCI Machine Learning Repository |
| **Download URL** | `https://archive.ics.uci.edu/dataset/447/condition+monitoring+of+hydraulic+systems` |
| **License** | CC BY 4.0 (Creative Commons Attribution) |
| **Citation** | Helwig, N., Pignanelli, E., & Schütze, A. (2015). "Condition Monitoring of a Complex Hydraulic System Using Multivariate Statistics." Proc. I2MTC-2015. |
| **Nature** | Real sensor data from a physical hydraulic test rig with systematically varied fault conditions |
| **Sensors** | Pressure (PS1-PS6), Motor power (EPS1), Volume flow (FS1-FS2), Temperature (TS1-TS4), Vibration (VS1), Efficiency factor (SE), Virtual sensors (CE, CP) |
| **Faults** | Cooler efficiency, valve switching lag, pump leakage, accumulator pressure — each with documented severity levels |
| **Size** | 2,205 cycles × 17 sensor channels at varying sample rates (1 Hz to 100 Hz) |

## Physics-Informed Synthetic Datasets

> [!IMPORTANT]
> The following datasets are **generated by physics-informed simulators** built as part of this project, because no suitable public real-world dataset exists for these specific subsystems. This is standard practice in PHM research (C-MAPSS itself is synthetic). Every record carries `is_synthetic_data: true` and the UI clearly labels these subsystems.

### 3. Landing Gear / Brakes — Synthetic

| Field | Value |
|---|---|
| **Generator** | `ml/landing_gear/synthetic_generator.py` |
| **Physics Model** | Brake energy dissipation: E = ½mv² × braking_fraction; temperature rise from specific heat; wear as cumulative thermal cycling |
| **Parameters** | Wheel velocity (0–250 kts), brake core temperature (50–900°C), brake pressure (0–3000 psi), aircraft mass (50–80 t), runway friction coefficient |
| **Classes** | nominal, light_wear, moderate_wear, heavy_wear, critical |
| **Size** | ~5,000 synthetic landing events |
| **Synthetic** | **YES** — clearly labeled in all outputs |

### 4. APU (Auxiliary Power Unit) — Synthetic

| Field | Value |
|---|---|
| **Generator** | `ml/apu/synthetic_generator.py` |
| **Physics Model** | EGT startup curve modeled as exponential rise to steady-state with controllable fouling/degradation parameter affecting time constant, steady-state temperature, and fuel efficiency |
| **Parameters** | EGT startup profile, fuel flow rate, turbine inlet temperature, fouling level (0–100%) |
| **Size** | ~200 synthetic APU units with varying degradation levels |
| **Synthetic** | **YES** — clearly labeled in all outputs |

### 5. ECS (Environmental Control System) — Synthetic

| Field | Value |
|---|---|
| **Generator** | `ml/ecs/simulator.py` |
| **Physics Model** | Parametrized reverse-Brayton-cycle: bleed air through primary/secondary heat exchangers, ACM compressor/turbine, water separator. Standard ideal-gas thermodynamic relations with constant specific heat. |
| **Key Feature** | Controllable heat exchanger fouling parameter (0–100%) with explicit cross-domain coupling to engine bleed air demand |
| **Size** | Generated on-demand for simulation scenarios |
| **Synthetic** | **YES** — clearly labeled in all outputs |

## NLP / Maintenance Log Data

### 6. Curated Maintenance Text Pairs

| Field | Value |
|---|---|
| **Source** | Manually curated from public aviation maintenance resources, FAA Advisory Circulars, and generic maintenance procedure texts |
| **Nature** | ~300 problem-description / recommended-action text pairs |
| **Usage** | TF-IDF + cosine similarity retrieval (Phase 1 stand-in for planned Phase 2 fine-tuned LLM) |
| **License** | Public domain / fair use educational materials |
| **Synthetic** | Partially curated — not a published dataset |
